## Development Environment

**Prerequisites**:
- .NET 8.0 SDK (or later)
- A compiler supporting C# 14 (Visual Studio 2022 17.12+, Rider 2024.3+, or .NET 9+ SDK)

**Clone and build**:

```bash
git clone https://github.com/RDCN-Community-Developers/RhythmToolkit.git
cd RhythmToolkit
dotnet restore
dotnet build
```

**Local NuGet package debugging**:

Sub-projects (RhythmDoctor, Adofai, etc.) reference the core library via NuGet. When debugging, you need to pack the core library into a local NuGet source first:

```powershell
# Pack in the RhythmBase project directory
dotnet pack -c Release -o ./nupkg

# Add local source (first time)
dotnet nuget add source ./nupkg --name local

# Re-pack after each core library modification
dotnet pack -c Release -o ./nupkg --force
```

> NuGet caching in sub-projects may prevent updates from taking effect. Run `dotnet nuget locals all --clear` to clear the cache, then restore again.

## Project Architecture

```
RhythmBase                         ← Core library
├── Global/                        ← Public interfaces, types, serialization infrastructure
│   ├── Components/                ← Color, EnumCollection, ITickTime, ILevel...
│   ├── Events/                    ← IEvent, IDurationEvent, IFileEvent, IForwardEvent...
│   ├── Converters/                ← MetadataJsonConverter, MemberConverter, TypeConverterRegistry...
│   └── Extensions/                ← LINQ queries, event navigation

RhythmBase.Generator              ← Source generator
└── Based on attribute declarations in AssemblyInfo.cs, auto-generates:
    ├── EventConverterMap          ← Event enum → MemberConverter routing table
    ├── EventTypeRegistry          ← Type ↔ enum bidirectional mapping + classification queries
    ├── EnumConverterExtensions    ← Enum TryParse / ToEnumString extension methods
    ├── TypeConverterRegistry      ← Shared type converter registry
    └── MemberConverter per event  ← Property-level serializer

RhythmBase.RhythmDoctor            ← Rhythm Doctor adapter
RhythmBase.Adofai                  ← A Dance of Fire and Ice adapter
RhythmBase.BeatBlock               ← BeatBlock adapter
RhythmBase.Rizline                 ← Rizline adapter
```

**Two serialization tracks**:

```
MetadataJsonConverter<T>     — Manages { } boundaries (Level, Settings, Row, etc.)
└── BaseEventConverter       — Routes to EventConverterMap based on "type" field

MemberConverter<T>           — Manages fields inside { } (individual event property read/write)
└── Concrete event converter — Auto-generated by source generator
```

## Adapting a New Game

### Quick Checklist

- [ ] Create project with `RootNamespace` set to `RhythmBase`, reference `RhythmBase` NuGet package
- [ ] Define event type enum, mark with `[JsonEnumSerializable]`
- [ ] Define time unit, implement `ITickTime<TickTime>`
- [ ] Define event interface (e.g., `IBaseEvent`) and event base class
- [ ] Define event subclasses, mark with `[JsonObjectSerializable]`
- [ ] Define level model `Level`, implement appropriate `ILevel` interface combination
- [ ] Create `AssemblyInfo.cs`, register `JsonConverterId` + `JsonConverterSourceType`
- [ ] Implement hand-written converters (`LevelConverter`, `SettingsConverter`, `BaseEventConverter`, etc.)
- [ ] Implement serialization methods (`FromFile`, `SaveToFile`, `FromZip`, etc.)
- [ ] Create `GlobalUsing.cs`
- [ ] All compilation passes with no errors

### Detailed Steps

See [Tutorial.md](docs/Tutorial.md#ii-implementing-a-new-level-type).

## Source Generator

### What the Generator Does

The source generator (`RhythmBase.Generator`) is a Roslyn Incremental Source Generator that scans attribute declarations in the project at compile time and automatically generates the following code:

```
Scan attributes in AssemblyInfo.cs
    ↓
┌─ JsonConverterId("RhythmDoctor")
│   → Generates TypeConverterRegistry.RhythmDoctor.g.cs
│     (shared type converter registry)
│
├─ JsonConverterSourceType(IBaseEvent, EventType, MemberConverter<>, "Type")
│   → Generates Converters.RhythmDoctor.g.cs
│     (MemberConverter for each event class + EventConverterMap routing table)
│
├─ JsonConverterLink(typeof(Color), typeof(ColorConverter.RgbaHex))
│   → Injected into TypeConverterRegistry
│
└─ Event classes marked with [JsonObjectSerializable]
    → Generates corresponding MemberConverter subclass
    → Generates EventTypeRegistry (type ↔ enum mapping)
    → Generates EnumConverterExtensions (TryParse / ToEnumString)
```

### What the Developer Needs to Provide

| Must provide | Description |
|---|---|
| Event enum | With `[JsonEnumSerializable]`, member names match event class names |
| Event interface | E.g., `IBaseEvent : IEvent<EventType, TickTime>` |
| Event base class | With `_extraData` dictionary and indexer |
| Event subclasses | With `[JsonObjectSerializable]`, `Type` property returns enum value |
| Fallback event | With `[JsonObjectSerializationFallback]` (globally unique) |
| `AssemblyInfo.cs` | Register `JsonConverterId` + `JsonConverterSourceType` |
| Hand-written converters | `LevelConverter`, `SettingsConverter`, `BaseEventConverter`, etc. |

| Auto-generated | Description |
|---|---|
| `MemberConverter<T>` per event | Handles event property read/write |
| `EventConverterMap` | Enum value → MemberConverter switch routing |
| `EventTypeRegistry` | Type ↔ enum bidirectional mapping + classification queries |
| `TypeConverterRegistry` | Converter registration for shared types (Color, RichLine, etc.) |
| `EnumConverterExtensions` | Enum `TryParse` / `ToEnumString` extension methods |

### Generated Artifacts

After compilation, generated files can be found in the `obj/` directory:

```
obj/Debug/net8.0/generated/RhythmBase.Generator/
├── Converters.RhythmDoctor.g.cs          ← Event converters + EventConverterMap
├── EnumConverters.RhythmDoctor.g.cs      ← Enum conversion extension methods
├── ConverterRegistry.g.cs                ← TypeConverterRegistry + FileMainEntryConverter
└── ClassEnumMap.g.cs                     ← EventTypeRegistry
```

> Generated files have a `// <auto-generated/>` header marker.

## Hand-Written Converters

The following compound types need hand-written converters because their serialization logic cannot be automatically inferred by the source generator:

| Converter | Responsibility | Reference |
|---|---|---|
| `LevelConverter` | Read/write entire level object (Settings, Rows, Decorations, Events, etc.) | `RhythmDoctor/Converters/LevelConverter.cs` |
| `SettingsConverter` | Read/write level settings | `RhythmDoctor/Converters/SettingsConverter.cs` |
| `BaseEventConverter` | Event type routing — scans the `type` field in JSON, dispatches to `EventConverterMap` | `RhythmDoctor/Converters/BaseEventConverter.cs` |

**Hand-written converter pattern**:

```csharp
// Inherit MetadataJsonConverter<T>, handle { } boundary
internal class LevelConverter : MetadataJsonConverter<Level>
{
    public override Level? Read(ref Utf8JsonReader reader, Type typeToConvert, MetadataJsonSerializerOptions options)
    {
        // Read fields inside { ... }
        // For sub-objects (Settings, Events), call corresponding converters
    }

    public override void Write(Utf8JsonWriter writer, Level value, MetadataJsonSerializerOptions options)
    {
        // Write fields inside { ... }
    }
}
```

**BaseEventConverter special pattern**:

```csharp
// Scan "type" field → call EventConverterMap.GetConverter(type) → delegate to MemberConverter
internal class BaseEventConverter : MetadataJsonConverter<IBaseEvent>
{
    public override IBaseEvent? Read(ref Utf8JsonReader reader, Type typeToConvert, MetadataJsonSerializerOptions options)
    {
        // 1. Scan JSON to find the "type" field value
        // 2. EventConverterMap.GetConverter(typeEnum) to get corresponding MemberConverter
        // 3. MemberConverter.WithReadSettings(_rs).ReadProperties(ref reader, options)
    }
}
```

## Code Conventions

### Namespaces

```
RhythmBase.{GameType}.{CategoryType}
```

- GameType: `RhythmDoctor`, `Adofai`, `BeatBlock`, `Rizline`
- CategoryType: `Components`, `Events`, `Converters`, `Constants`, `Extensions`, `Utils`

### File Organization

```
RhythmBase.{Game}/
├── AssemblyInfo.cs                    ← Source generator registration entry point
├── GlobalUsing.cs                     ← Global using directives
├── {Game}/
│   ├── GlobalUsing.cs                 ← Game sub-namespace global using
│   ├── Enums.cs                       ← Event enums + game-specific enums
│   ├── Components/
│   │   ├── Level.cs                   ← Level model (partial class)
│   │   ├── Level.SerializeMethods.cs  ← Serialization methods (FromFile/SaveToFile, etc.)
│   │   ├── TickTime.cs                ← Time unit
│   │   ├── Settings.cs                ← Level settings
│   │   └── ...
│   ├── Events/
│   │   ├── IBaseEvent.cs              ← Event interface
│   │   ├── BaseEvent.cs               ← Event base class
│   │   ├── ForwardEvent.cs            ← Fallback compatibility event
│   │   └── ... (concrete event classes)
│   ├── Converters/
│   │   ├── BaseEventConverter.cs      ← Event routing converter
│   │   ├── LevelConverter.cs          ← Level converter
│   │   └── ...
│   ├── Constants/
│   │   └── Constants.cs               ← Predefined constants
│   ├── Extensions/
│   │   └── LinqExtensions.cs          ← LINQ extensions
│   └── Utils/
│       └── BeatCalculator.cs          ← Beat calculator (if applicable)
└── Global/
    └── Components/Vector/             ← Project-level geometry type extensions (if needed)
```

### Event Class Conventions

- All events must be `record` types (supporting `with` expressions and value equality)
- Each event must have an `override EventType Type { get; }` property returning the corresponding enum value
- Event base class must contain `internal Dictionary<string, JsonElement> _extraData` and an indexer
- Use `[JsonObjectSerializable]` to mark events that need auto-generated converters
- Use `[JsonObjectNotSerializable]` to mark fallback types like `ForwardEvent`
- Use `[JsonObjectSerializationFallback]` to mark one fallback event class (globally unique)

### TickTime Conventions

- Must be a `struct` implementing `ITickTime<TickTime>`
- Must support construction from `float`, `TimeSpan`
- Must implement `IComparable<TickTime>` and comparison operators
- Recommended: implement association/disassociation mechanism with `BeatCalculator`

## Attribute Reference

### Type-Level Attributes

| Attribute | Target | Description |
|---|---|---|
| `[JsonEnumSerializable]` | Enum | Marks an enum for string-based serialization. Default PascalCase; `[JsonEnumSerializable(false)]` for camelCase |
| `[JsonObjectSerializable]` | Class/Struct | Auto-generates serializer and EventTypeRegistry mapping |
| `[JsonObjectHasSerializer(typeof(C))]` | Class/Struct | Has custom serializer, still needs mapping to EventTypeRegistry |
| `[JsonObjectNotSerializable]` | Class/Struct | No serializer or mapping needed (e.g., `ForwardEvent`) |
| `[JsonObjectSerializationFallback]` | Class/Struct | Fallback model for unknown types (one per `JsonConverterSourceType` declaration) |

### Property-Level Attributes

| Attribute | Target | Description |
|---|---|---|
| `[JsonAlias("name")]` | Property/Field | Alias used in JSON (default auto camelCase) |
| `[JsonIgnore]` | Property/Field | Ignored during serialization/deserialization |
| `[JsonCondition("$&.Prop != value")]` | Property/Field | Conditional write. `$&` = object itself, `$r` = read settings, `$w` = write settings |
| `[JsonTime(JsonTimeType.Milliseconds)]` | Property/Field | TimeSpan serialized as milliseconds or seconds |
| `[JsonConverter(typeof(C))]` | Property/Field | Use specified converter |
| `[JsonDefaultSerializer]` | Property/Field | Use default enum serializer (numeric instead of string) |

### Assembly-Level Attributes

| Attribute | Description |
|---|---|
| `[JsonConverterId("GameType")]` | Declares game type namespace ID, used to generate `TypeConverterRegistry`, `EnumConverterExtensions`, etc. |
| `[JsonConverterSourceType(I, E, C, P)]` | Declares event type system. `I` = event interface, `E` = event enum, `C` = converter base class (open generic), `P` = enum property name |
| `[JsonConverterLink(Type, Converter)]` | Links a shared type (e.g., `Color`) to a specified converter (e.g., `ColorConverter.RgbaHex`) |

### Attribute Condition Expression Syntax

`[JsonCondition]` expressions support the following placeholders:

| Placeholder | Replaced with |
|---|---|
| `$&` | Current object instance (`value`) |
| `$r` | Read settings (`_rs`) |
| `$w` | Write settings (`_ws`) |

```csharp
// Only write when Swing is not the default value
[JsonCondition($"$&.{nameof(Swing)} is not 1f and not 0f")]
public float Swing { get; set; }
```
